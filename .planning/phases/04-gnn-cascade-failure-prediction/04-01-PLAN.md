Create backend/ml/gnn_train.py as a self-contained offline training script.

**Grid topology facts** (from backend/state.py):
- 24 nodes (NODE-01 to NODE-24)
- 39 directed edges (see EDGES list in backend/state.py)
- Node types: generator(4), substation(8), transformer(8), junction(4)
- Node index = int(node_id.split('-')[1]) - 1  (i.e., "NODE-01" → index 0)

**GCN architecture** (2-layer GCN, small enough to train in <30 seconds on CPU):
```python
class CascadeGCN(torch.nn.Module):
    def __init__(self, in_channels=5, hidden=32, out_channels=1):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden)
        self.conv2 = GCNConv(hidden, out_channels)

    def forward(self, x, edge_index):
        x = F.relu(self.conv1(x, edge_index))
        x = F.dropout(x, p=0.3, training=self.training)
        return torch.sigmoid(self.conv2(x, edge_index))
```

**Node features** (5 features per node):
1. load / 100.0 (normalized)
2. abs(frequency - 60.0) / 1.0 (normalized deviation)
3. 1.0 if anomalous else 0.0 (anomaly flag)
4. node type encoding: generator=1.0, substation=0.75, transformer=0.5, junction=0.25
5. degree / max_degree (graph connectivity, normalized)

**Synthetic training data generation** (generate 2000 samples):
For each sample:
1. Pick 1-3 random "fault seed" nodes (simulate XGBoost anomaly detection output)
2. For each node in the graph, label it as "cascade at-risk" (1) if:
   - It IS a fault seed node
   - OR it is within graph distance 1-3 hops from a fault seed AND random(0,1) < 0.7 * (1 / distance)
   (This creates realistic propagation: close neighbors more likely to cascade)
3. Add noise: flip labels with probability 0.05 (5% noise for robustness)
4. Perturb features: add random noise N(0, 0.05) to numeric features

Build edge_index as a torch.long tensor from the 39 EDGES (bidirectional — add both directions).

**Training**:
- Loss: BCELoss (binary classification per node)
- Optimizer: Adam, lr=0.01
- Epochs: 100
- Print loss every 20 epochs

**Save**: `torch.save(model.state_dict(), 'backend/ml/gnn_model.pt')`

Run the script at the end: `if __name__ == '__main__': train()`

The script should work when run as `python -m backend.ml.gnn_train` from the project root.

```python
import os, torch, heapq
from torch_geometric.nn import GCNConv
import torch.nn.functional as F
from backend.ml.gnn_train import CascadeGCN, build_edge_index, NODE_IDS, NODE_TYPE_ENC

_MODEL_PATH = os.path.join(os.path.dirname(__file__), 'gnn_model.pt')
_MODEL: CascadeGCN | None = None

def _load_model() -> CascadeGCN:
    global _MODEL
    if _MODEL is None:
        m = CascadeGCN()
        m.load_state_dict(torch.load(_MODEL_PATH, map_location='cpu'))
        m.eval()
        _MODEL = m
    return _MODEL
```

**predict_cascade() signature**:
```python
def predict_cascade(
    anomalous_node_ids: list[str],
    node_states: dict,   # NODE_STATES dict from backend/state.py
    edges: list[dict],   # EDGES list from backend/state.py
) -> dict:
```

**Return shape** (the API contract Plan 02 depends on):
```python
{
    "cascade_chain": [
        {
            "node_id": "NODE-07",
            "confidence": 0.87,           # GCN output probability
            "time_to_cascade_min": 4.2,   # derived: (1 - confidence) * 10 clamped to [1, 10]
            "hop_distance": 1             # BFS distance from nearest anomalous node
        },
        ...  # ordered by confidence desc
    ],
    "rerouting_path": ["NODE-01", "NODE-05", "NODE-13"],  # [] if no path found
    "rerouting_summary": "via Lincoln Park Substation → River North Transformer, +12% margin"
}
```

**GCN forward pass**:
1. Build feature tensor from node_states (same 5 features as training)
2. Mark anomalous_node_ids with anomaly flag = 1.0 in feature tensor
3. Run model forward pass → per-node confidence scores
4. Filter: only include nodes with confidence > 0.35 in cascade_chain
5. Exclude anomalous_node_ids themselves from the chain (they are already failed)
6. Compute hop_distance via BFS from anomalous nodes through EDGES graph
7. Derive time_to_cascade_min = max(1.0, (1.0 - confidence) * 10.0) — lower confidence = sooner cascade (more urgency communicated)
8. Sort cascade_chain by confidence descending

**Dijkstra rerouting** (clean separation from GNN):
- Build adjacency from EDGES with capacity as inverse weight (higher capacity = lower cost)
- Remove all nodes with confidence > 0.75 from the traversable graph (high-risk nodes avoided)
- Find shortest path between any two non-anomalous, non-high-risk nodes on opposite sides of the fault
  - "Source" = first non-risky generator node
  - "Target" = first non-risky substation/junction node at graph distance >= 2 from epicenter
  - If no valid pair found, return rerouting_path = []
- rerouting_summary: "via {node2.name} → {node3.name}, +{margin}% margin" where margin = int((target_capacity - min_edge_capacity) / target_capacity * 100)

**Exception guard**: wrap entire function body in try/except, return `{"cascade_chain": [], "rerouting_path": [], "rerouting_summary": ""}` on any error — never crash the inference loop.

Import gnn_train.CascadeGCN using deferred import pattern (same as xgboost inference uses deferred import) — import at the top of the file is fine here since gnn_train has no circular dependency risk with the rest of the backend.
